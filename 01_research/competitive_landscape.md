# 경쟁 지형도 (Competitive Landscape)

> Phase 1 초안. 사용자 인터뷰 후 갱신.
>
> **관찰 기준**: 2026-04-24 시점의 각 서비스 공개 정보(공식 홈페이지·문서·공개 리뷰)를 기반으로 작성했습니다. 각 제품은 적극적으로 개선 중이며, 이 문서의 내용은 특정 시점의 스냅샷임을 유의해 주세요. 잘못된 내용은 이슈로 알려 주시면 감사하겠습니다.

## 분류 프레임

논문을 둘러싼 사용자 여정을 네 단계로 쪼개면:

```
발견(Discover) → 저장(Organize) → 이해(Understand) → 행동(Act)
```

기존 도구들은 이 중 **하나나 둘**을 잘하지만, **전체를 잇는 제품**은 공백. Synapse의 포지셔닝은 **"Understand → Act"** 다리.

## 직접 경쟁 / 인접 도구

### 1. SciSpace (구 Typeset)
- **무엇**: 논문 PDF 대화·요약·표 추출, Literature Review 자동화.
- **잘하는 것**: 논문 대화 UX 완성도, 다국어 번역·요약.
- **Synapse와 겹치지 않는 범위**: 답변의 원문 인용 하이라이트는 제한적이며, 읽고 난 뒤의 행동(구현·공유·후속)까지는 제품 범위 밖으로 관찰됨.
- **시사점**: 이해 단계가 주력. Synapse는 "이해 이후의 행동"을 차별 영역으로 설정.

### 2. Elicit
- **무엇**: 연구 질의에 대해 다수 논문에서 증거 추출, 표로 정리.
- **잘하는 것**: Systematic review 워크플로우, 논문 집합 필드 추출.
- **Synapse와 겹치지 않는 범위**: 단일 논문을 깊이 "읽는" 몰입형 UX는 주된 범위가 아님.
- **시사점**: 서베이/리뷰 맥락의 사용자에게는 **대체가 아닌 보완**. 연동 가능성 탐색.

### 3. Consensus
- **무엇**: 자연어 질문 → 논문 요지 기반 답변.
- **잘하는 것**: "이 주제에 대한 학계의 공통 견해"를 빠르게 파악.
- **Synapse와 겹치지 않는 범위**: 단일 논문 내부를 깊이 읽어 들어가는 경험은 제공 범위 밖.
- **시사점**: 탐색 단계 → 단일 논문 정독으로의 **연결 접점**.

### 4. NotebookLM (Google)
- **무엇**: 여러 소스(PDF·노트·링크) 묶어 Q&A·오디오 요약 생성.
- **잘하는 것**: 다중 소스 종합, 오디오 요약 품질, Gemini 장문 컨텍스트.
- **Synapse와 겹치지 않는 범위**: 논문 특유 구조(수식·그림·참고문헌 그래프) 처리, 행동 연계는 주력이 아닌 것으로 관찰. 일부 기관·개인은 Google 생태계 외 대안을 선호.
- **시사점**: 기능 중복 영역이 가장 큰 도구. **논문 특화 깊이 + 다음 스텝**에서 차별화 가능성 탐색.

### 5. ChatPDF / 유사 제품 (일반 PDF 챗)
- **무엇**: 임의 PDF 업로드 → 대화.
- **잘하는 것**: 진입 장벽이 매우 낮음, 범용.
- **Synapse와 겹치지 않는 범위**: 과학 논문 구조(섹션·수식·참고문헌) 최적화, 학술적 인용 품질은 범용 제품이라 깊이 있게 다루지 않음.
- **시사점**: 사용자의 "첫 시도" 선택지. Synapse는 **과학 도메인 정확도**로 선택받기.

### 6. Semantic Scholar + Connected Papers / ResearchRabbit
- **무엇**: 논문 검색·인용 그래프·관련 논문 탐색.
- **잘하는 것**: 발견 단계의 사실상 기준 도구. Semantic Scholar는 공개 API 제공.
- **Synapse와 겹치지 않는 범위**: 개별 논문 내부 이해·노트·행동 연계는 설계 범위 밖.
- **시사점**: **통합 대상(Partner)**. Synapse의 인용 그래프·관련 논문 기능의 데이터 소스로 활용.

### 7. Zotero / Mendeley / Paperpile
- **무엇**: 레퍼런스 매니저.
- **잘하는 것**: 저장·인용 스타일·기기 간 동기화·커뮤니티(Zotero)·문서 편집기 연동.
- **Synapse와 겹치지 않는 범위**: AI 기반 이해·웹 리서치 통합은 주된 범위가 아님.
- **시사점**: Synapse는 **이해·행동**에 집중하고, 레퍼런스 관리는 **연동**(Zotero import/export)으로 활용.

### 8. Explainpaper / Scholarcy
- **무엇**: 선택한 부분 설명 / 요약 카드 생성.
- **잘하는 것**: 가볍고 즉각적인 UX.
- **Synapse와 겹치지 않는 범위**: 단일 액션이 주력이며, 장기 세션·지식 축적은 설계 범위 밖.
- **시사점**: Synapse의 "이해 조력자" 기능의 한 조각. 우리는 **맥락·사용자 수준·인용**까지 함께 제공하는 것이 차별.

## Synapse 차별점 가설

1. **Understand → Act 다리**: 읽고 난 뒤의 행동(구현·공유·후속 리서치)까지 제품 안에서.
2. **사용자 수준에 맞춘 이해 보조**: beginner/practitioner/expert 프리셋.
3. **인용 기반 신뢰**: 모든 답변에 원문 포인터 필수. 할루시네이션 억제.
4. **기존 도구와의 연동**: Zotero·Semantic Scholar를 **대체**하지 않고 **잇는다**.

## 리스크 / "정말 차별화되는가"

- NotebookLM이 행동 기능을 강화할 경우 기능 중복이 확대됨 → **논문 도메인 깊이와 응답 속도**로 방어.
- SciSpace가 다음 스텝 기능을 확장할 경우 중복 발생 → **인용 신뢰성·세그먼트 깊이**로 방어.
- 레퍼런스 매니저 진영이 AI 통합을 강화할 경우 중복 발생 → **독립성·데이터 프라이버시 정책**을 명시적 가치로 제시.

## 인터뷰에서 검증할 것

- 사용자가 실제로 위 도구를 **병용**하는지, **대체**하는지
- 각 도구에서 어디서 막히는지 (포기 지점)
- "다음 스텝"에 대한 현재 워크어라운드
- 가격·프라이버시·기관 계정 의존도
