## 2. 공지문 템플릿

### 2.1 한국어

> **[연구 참여자 모집] 논문 읽기 경험 인터뷰 (60분 / 사례금 지급)**
>
> 안녕하세요. 저희는 연구 논문을 더 잘 이해하고 다음 행동으로 연결해 주는 도구를 만드는 팀입니다. 여러분이 **실제로 논문을 어떻게 읽고, 어디서 막히는지** 듣고 싶어요.
>
> - 소요 시간: 60분 (온라인)
> - 사례: (금액 결정 후 기입, 예: 문화상품권 5만원)
> - 조건: 지난 4주 내에 연구 논문을 1편 이상 읽으신 분
> - 형식: 반구조화 인터뷰, 녹화는 동의 시에만
>
> 참여 의향이 있으시면 아래 2분짜리 설문을 작성해 주세요 → [스크리너 링크]
>
> (문의: [연구 담당자 이메일])

## 3. 스크리닝 설문 (Google Form / Typeform 권장)

### 기본 정보
1. 이름 (익명화됩니다)
2. 이메일
3. 소속·역할 한 문장 (예: "KAIST EE 박사과정 2년차", "스타트업 AI 엔지니어")
4. 지리적 위치 (국가/도시)

### 세그먼트 분류 — 가장 가까운 한 가지 선택
- [ ] 대학원생 (석사·박사, 서베이/리뷰 작성 경험)
- [ ] AI/ML 엔지니어 (논문 → 구현 빈번)
- [ ] 학습자/비전공자 (직무 외 논문도 읽음)
- [ ] 산업 리서처 (보고서·시장조사·전략)
- [ ] 기타 (자유 기입)

### 논문 읽기 빈도·맥락
5. 최근 4주 동안 논문을 몇 편 정도 읽으셨나요? (0 / 1–3 / 4–10 / 10+)
6. 주로 어떤 분야의 논문인가요? (자유 기입)
7. 논문을 읽는 주된 목적은? (복수 선택)
   - [ ] 연구 주제 서베이
   - [ ] 방법 구현/재현
   - [ ] 랩미팅·리뷰 준비
   - [ ] 학습/공부
   - [ ] 보고서·의사결정
   - [ ] 기타

### 현재 도구
8. 논문을 읽을 때 사용하는 도구를 모두 선택 (복수 선택)
   - [ ] PDF 뷰어만 (Acrobat/Preview)
   - [ ] Zotero / Mendeley / Paperpile
   - [ ] SciSpace / ChatPDF / NotebookLM 등 AI 도구
   - [ ] Obsidian / Notion 등 노트
   - [ ] Semantic Scholar / Elicit / Consensus
   - [ ] 기타 (자유 기입)

### 동의·일정
9. 60분 인터뷰에 참여 의향 (예/아니오)
10. 녹화·녹음 동의 여부 (예/아니오/"인터뷰 시 결정")
11. 선호 시간대 (평일 오전/오후/저녁 · 주말)
12. 타임존

## 4. 선정 기준

**우선**
- 주 세그먼트에서 골고루 (각 1–2명 목표)
- "읽다가 포기"·"읽고 무엇을 할지 막힘" 경험이 있는 사람
- 기존 AI 도구 **사용 경험자**와 **미경험자** 섞기

**제외**
- 현재 유사한 논문 읽기 도구를 직접 개발 중이신 분 (관점 편향 가능성으로 제한)
- 참여 동의 철회, 또는 일정 조율 2회 이상 실패
