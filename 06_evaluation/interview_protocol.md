# 인터뷰 프로토콜 — 정성 평가

## 0. 공통 원칙
- **유도 질문 금지**: "이 기능 유용할까요?" 류 X. 대신 **최근 실제 경험**을 단계별로 재구성하게 한다.
- **답변이 아닌 행동을 듣는다**: "어떻게 하셨나요?" > "어떻게 하고 싶으세요?"
- **침묵을 인내한다**: 2–3초 빈 공간이 가장 풍부한 답을 가져온다.
- **사실과 해석을 분리한다**: 노트에 `[observation]` 과 `[interpretation]` 태그.

---

## 1. Discovery 인터뷰 (Phase 1)

### 목적
- 논문을 읽는 **실제 맥락**, 사용 도구, 포기 지점 이해
- "읽고 난 뒤의 간극"이 얼마나 강한 불만인지 검증
- 가설 세그먼트 중 **가장 뜨거운 세그먼트** 식별

### 대상 모집
- 5–8명, 세그먼트 다양성 확보 (대학원생 · AI/ML 엔지니어 · 학습자 · 산업 리서처 각 1–2명)
- 지난 4주 내 논문을 1편 이상 읽은 사람

### 구성 (60분)
| 시간 | 블록 | 내용 |
|------|------|------|
| 5분 | Warm-up | 자기소개, 평소 논문 읽는 빈도·이유 |
| 15분 | 최근 경험 | 최근 읽은 논문 1편을 단계별로 재구성: 어디서 발견 → 왜 읽음 → 어떻게 시작 → 어디서 막힘 → 끝까지 갔나 → 그다음? |
| 10분 | 도구 | 현재 쓰는 도구, 좋은 점·불만 |
| 15분 | 페인 포인트 | "읽는 중"·"읽은 후" 각각 가장 힘든 순간 3가지 |
| 10분 | 희망 사항 | 마법 지팡이가 있다면 **결과**가 어떻게 바뀌길 원하는지 (기능 X) |
| 5분 | 마무리 | 후속 참여 의향, 소개해 줄 사람 |

### 금지 질문
- "우리 제품을 쓰실 건가요?"
- "이 기능이 있다면 유용할까요?"
- "어떤 기능이 추가되면 좋을까요?" (사용자는 기능을 잘 말하지 못함)

### 핵심 발견 질문 (Key Questions)
- 논문을 읽고 **그다음 행동을 한 번이라도** 해본 적이 있는가? 그게 뭐였고 어떻게 촉발됐는가?
- 읽다가 **그만 둔 논문**이 있다면 왜였는가?
- 동료/공동연구자에게 논문 내용을 **어떻게** 공유하는가?

## 2. 사용성 인터뷰 (Phase 5)

### 목적
- MVP 프로토타입으로 **실제 과업 수행** 시 어려움·성공 포인트 관찰
- 북극성 가설(insight-to-action) 검증

### 대상
- 8–10명
- Discovery 참여자 중 재활용 가능한 인원 + 신규
- 각 세그먼트 최소 1명

### 구성 (60–75분)
| 시간 | 블록 | 내용 |
|------|------|------|
| 5분 | Warm-up | 현재 상황 · 오늘 다룰 논문 1편 |
| 15분 | **과업 1** | 새 논문 임포트 → 핵심 이해 (think-aloud) |
| 15분 | **과업 2** | 특정 용어/수식에서 막힘 → 에이전트에 질문 → 만족할 때까지 |
| 15분 | **과업 3** | "이 아이디어를 내 프로젝트에 적용" 시나리오 → 다음 스텝 제안 수용 |
| 5분 | SUS | 10문항 (1–5 점) |
| 10분 | 오픈 피드백 | 기대 대비 실제, 경쟁 도구 비교 |

### 기록
- 참여자 동의 시 화면·음성 녹화
- 노트는 `06_evaluation/interview_notes/YYYY-MM-DD_P[번호].md` 에 저장
- 익명화 코드: `P01`, `P02`, …
- 개인 식별 정보(소속·실명·이메일)는 **별도 파일** 로 관리, 공개 저장소에 커밋 X

## 3. 분석

**Thematic Analysis (Braun & Clarke 2006)**

1. **Familiarization** — 전사본 전체 읽기
2. **Coding** — 의미 단위마다 코드 부여 (open coding)
3. **Themes** — 코드를 그룹핑해 테마 도출
4. **Review** — 테마가 데이터를 충실히 설명하는지 재확인
5. **Define & Name** — 테마 이름 확정
6. **Write-up** — `06_evaluation/reports/phaseN_qualitative_summary.md`

### 품질 확보
- 솔로 분석일 경우 **24시간 간격 재코딩** → 코딩 일관성 점검
- 2인 이상이면 **inter-rater agreement** 샘플링

## 4. 윤리·프라이버시

- **동의서** — 참여 전 서면 동의 (`06_evaluation/consent_form_template.md` 작성 예정)
- 녹취 목적·보관 기간·폐기 정책 명시
- 민감 내용(본인 연구) 취급: 업로드한 논문·메모는 분석에 사용하지 않음 (관찰 대상은 **인터랙션**)
- 참여 보상 기준 명시
