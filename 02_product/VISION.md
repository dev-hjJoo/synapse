# Synapse — Product Vision

## 한 줄 요약
논문을 **읽는 도구**가 아니라, **이해의 스파크를 다음 행동으로 잇는** 플랫폼.

## 배경: 왜 지금인가
- 논문·프리프린트 생산량이 모든 분야에서 지수적으로 증가 중.
- LLM 덕분에 **장문 문서의 이해·요약·대화**가 처음으로 현실적인 수준.
- 그러나 대부분의 도구는 "읽기 전" 또는 "읽는 중" 에서 멈추고, **"읽고 난 뒤의 행동"** 까지 이어주지 못함.

## 문제
연구자·학습자·산업 실무자 모두 비슷한 불만을 가진다. 현재의 도구 지형은 세 영역으로 쪼개져 있다.

1. **저장·하이라이트**: Zotero, Mendeley → 읽고 기억하는 것까지만
2. **요약·Q&A**: SciSpace, Consensus, ChatPDF → 이해 시점에서 끝
3. **검색·발견**: Semantic Scholar, Elicit → 찾기까지만

읽은 뒤 **"그래서 나는 뭘 하지?"** 의 간극이 비어있다.

## 우리가 채우는 간극
- **이해 보조** — 어려운 수식·용어·맥락을 **사용자의 배경지식** 기준으로 설명. 일반 독자에겐 직관을, 전문가에겐 원문의 미묘한 주장까지.
- **연결** — 인용된 논문, 후속 논문, 구현 코드, 블로그, 영상을 **한 자리에서** 탐색. 인용 관계를 따라가는 마찰 최소화.
- **행동** — "이 아이디어를 내 프로젝트에 적용해 볼래?" "구현 스캐폴딩 만들까?" "팀 리뷰용 슬라이드로 변환?" 같은 **다음 스텝**을 제안하고 실제로 돕는다.

## 북극성 지표 (가설, Phase 1 검증)
**주간 활성 사용자당 "insight-to-action" 전환 수** — 논문을 읽은 뒤 다음 행동(구현 시작, 공유, 후속 질의, 추가 자료 저장)으로 이어진 건수.

## 초기 타겟 (넓게)
범용적인 연구 논문 리더 전반. 내부 가설 세그먼트:

| 세그먼트 | 핵심 동기 | 예상 핵심 가치 |
|----------|-----------|----------------|
| 대학원생 | 서베이·랩 미팅·리뷰 | 빠른 맥락 파악, 인용 그래프 탐색 |
| AI/ML 엔지니어 | 최신 기법 구현 | 재현 가능한 구현 스캐폴딩 |
| 학습자·비전공자 | 깊이 있는 입문 | 사용자 수준 맞춤 설명 |
| 산업 리서처 | 보고서·의사결정 | 출처 인용, 요약 아티팩트 |

Phase 1 인터뷰로 **가장 강한 불만·의지를 가진 세그먼트**를 식별해 우선순위 조정.

## 비-목표 (Non-Goals, 초기)
- 논문 **작성·저술** 도구 (Overleaf 대체 X)
- 풀세트 레퍼런스 매니저 (Zotero 대체 X — 연동은 할 수 있음)
- 소셜 네트워크 / 독서 클럽 (공유는 산출물 기반으로만)
- 폐쇄형 저널 PDF의 재배포

## 핵심 가정 (검증 필요)
1. 사용자는 일회성 요약보다 **대화형 이해**를 원한다.
2. "다음 스텝" 제안은 단순한 포인팅이 아니라 **실행을 시작**시켜 줄 때 가치가 있다.
3. 범용성은 특정 세그먼트의 깊이를 희생할 만큼 중요하다 — 세그먼트별 **모드/프리셋**이 해법일 수 있다.
4. 인용·출처 기반 응답이 **신뢰의 핵심**이다. 할루시네이션은 초기 사용자에게 치명적.

## 리스크
- **AI 할루시네이션**이 과학 도메인 신뢰를 깎음 → 인용 기반 응답 · 출처 하이라이트 필수
- **논문 저작권** → arXiv 등 오픈 소스 우선, 유료 저널은 사용자 업로드 기반 + 저장/공유 정책
- **비용 구조** → 세션당 LLM 비용이 ARPU를 넘을 위험 → 모델 계층화·캐싱·검색 전략 필요
- **범용성 vs 깊이**의 트레이드오프 → 잘못하면 "아무에게도 사랑받지 못하는" 제품이 될 수 있음

## 성공 시나리오 (12개월)
- 수천 명의 주간 활성 독자가 Synapse에서 논문을 읽는다.
- 그중 30% 이상이 읽은 뒤 **행동(구현·공유·후속 리서치)** 으로 이어진다.
- 사용자 NPS ≥ 40. "이제 이거 없이는 논문 못 읽겠다"는 피드백이 반복된다.
- 인용 정확도·답변 신뢰도가 과학 커뮤니티에서 긍정적으로 언급된다.
